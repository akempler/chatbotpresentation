Conversation Design

Conversation Types

Flat Conversations

Easiest. Similar to an FAQ

Tree Based Conversations

More complex. Threaded/hierarchical conversations.

Tree Based allows a user to drill down to get more information. It also allows the chatbot to ask questions to fill slots.

Tone

What should it "sound" like?

Intents

What are the things a user wants to accomplish?

...The chatbot can only reply to intents that have been identified.

Utterances

What can a user say that will trigger each intent?

Using analytics and training we can adjust the utterances to increase the effectiveness of the bot.

Context

Understanding where a user is in a conversation.

Critical for tree based conversations.

Slots

What information do we need to capture from the user to move to the next part of the conversation.

Also relevant specifically to tree based conversations.

Fulfillment

Retrieve data from an external source.

For example, a list of options (such as locations) that are stored in an external database or available via an api. Or another example might be to validate a user account.

Conversation Repair

Get a user back into the conversation.

Escalation
